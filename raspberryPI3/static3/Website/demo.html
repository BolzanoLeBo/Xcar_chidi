X-Car

At the X-Car Project, our mission is to make life more accessible for visitors and students at INSA who face mobility challenges or need assistance with heavy items on campus. Our core mission is simple: providing transparent and safe transportation support. Therefore, we made a car that can follow a user and that explains each decision it makes.

Components

X-CAR is a small car in the one a lot of electronic devices have been added. We can find the following equipment:

  • 6 Ultrasonic sensors: 3 at the rear and 3 at the back of the car
  • A Lidar in the back
  • A Camera just next to the Lidar
  • 2 STM32 Bords, a RasberryPi and a JetsonNano to process all the different values from the sensors

Human Machine Interface (HMI)

In addition, we have built a web-application in order to use the vehicle. It has three diffenrent menus:

  • Home

    Enables the user to choose between the different modes available, as well as providing visual feedback during the execution of the modes.

  • State Machine

    The user will find all the available modes represented in blue, the mode they are in represented in green and the previous state in light green in the figure.
    Note that unavailable states are shown in gray

  • Terminal

    The terminal tab gives the user access to the vehicle's entire history, detailing the modes the user has been in and the current mode followed by the actions that activated the mode change.

What are the different modes ?

  1. Emergency Mode

    This mode is reachable from the HMI. The car is stopped and cannot acces to the other modes until the user allows it.

  2. Security Mode

    When the car encounter obstacles or a faillure the system goes in this mode. Then the car will be able to jump back the the original mode if the situation is resolved.

  3. Manual Mode

    Through the HMI, the user can drive the car with the joystick.

  4. Tracking Mode

    This is the bigger mode of our system. In this mode, the car will follow the user at a 2 meters distance The following steps describe how to use this mode:

    • Step 1: Take a picture of the user
    • Step 2: Strart the tracking

    In this mode the car is able to detect obstacles (humans or material) and to track the user at a fixed distance. The car is olso able to avoid obstacles.

  5. Idle Mode

    This mode is a home menu from the HMI point of view where the car is not moving. The systeme is waiting for a mode instruction.

Demonstration

  1. Manual Mode

    This mode allows the user to control the car using the joystick from the web interface. It also incorporates various features developed just below.

    1. Obstacle Detection

      The car is capable of detecting obstacles at 0.7m and enters a safety mode, visible on the interface. In this safety mode, the interface indicates the reason, such as obstacle detection in this demonstration. The vehicle is no longer able to move in the direction of the obstacle within the detection distance.

    2. System Checking

      A system integrity check program determines if a sensor is faulty. In this case, the system enters a security mode until the issue is resolved.

    3. Connection Loss

      When the connection is lost, the car returns to Idle Mode and waits for a new connection.

  2. Tracking Mode

    This mode allows the user to be followed by the X-Car at a distance of 1.7m. Several features have been implemented in this mode, including those mentioned earlier.

    1. User Initialization

      To track only the user, an initialization is required. This is done by taking a photo by pressing the init button on the interface.

    2. User Tracking + Emergency Stop

      Once initialized, tracking can begin. The car then follows the user at a distance of approximately 1.70m.
      If the user wants to stop the car due to incorrect behavior, the emergency stop button on the interface can be used to halt the vehicle.

    3. User Tracking in an Environment with Multiple People

      Thanks to the initialization, the car is able to recognize the user and therefore only follows them. If a person is in the field of view but does not match the user, they will be treated as an obstacle and not as a potential user.
